SECTION 1 – DNS SERVER

1.1 DNS Health Validation

# Check DNS server health
dcdiag /test:dns /v /e
Resolve-DnsName dc01.domain.local
Get-DnsServerDiagnostics
Get-DnsServerStatistics

# Check zones
Get-DnsServerZone
Get-DnsServerResourceRecord -ZoneName "domain.local" -RRType A

# Check forwarders
Get-DnsServerForwarder

# Flush DNS cache
Clear-DnsServerCache
Clear-DnsClientCache

1.2 DNS Troubleshooting Methodology

  1. Client: ipconfig /all — verify DNS server pointing to DC
  2. Client: nslookup domain.local — test resolution
  3. Client: nslookup -type=SRV _ldap._tcp.dc._msdcs.domain.local — test SRV records
  4. Server: Check DNS service running, zones loaded, forwarders configured
  5. Server: Check scavenging settings — stale records causing conflicts
  6. Verify: AD-integrated zone replication between DCs

1.3 Critical DNS Rules for MSP

  • Domain-joined machines MUST point to internal DNS (DC) — NEVER ISP DNS
  • DCs should point to themselves or partner DC for DNS — never external
  • Conditional forwarders for multi-domain/trust scenarios
  • Enable DNS debug logging for intermittent issues: Set-DnsServerDiagnostics -All $true
  • Scavenging: Enable on ONE DC only, 7-day no-refresh + 7-day refresh interval

SECTION 2 – DHCP SERVER

2.1 DHCP Health Check

# Check DHCP server status
Get-DhcpServerv4Statistics
Get-DhcpServerv4Scope
Get-DhcpServerv4Lease -ScopeId 192.168.1.0

# Check scope utilization (alert at 80%+)
Get-DhcpServerv4ScopeStatistics

# Check failover status
Get-DhcpServerv4Failover

# Authorize DHCP in AD
Get-DhcpServerInDC

2.2 DHCP Best Practices

  • Always use DHCP Failover (Hot Standby or Load Balance) — never single point
  • Exclude static IP ranges (servers, printers, network devices)
  • Set lease duration: 8 hours for wireless, 1-4 days for wired
  • Configure DHCP options: DNS servers (option 6), domain name (option 15), default gateway (option 3)
  • Enable conflict detection (1-2 attempts)
  • Backup DHCP database regularly

2.3 DHCP Troubleshooting

  • 169.254.x.x address: Client not reaching DHCP. Check network path, DHCP relay (ip helper-address on router)
  • Scope exhausted: Expand range, reduce lease time, clean stale leases
  • Duplicate IPs: Enable conflict detection, check for rogue DHCP: dhcploc.exe
  • Failover not replicating: Check partnership state, force replication

3.3 Common File Server Issues

  • Access denied: Check share permissions AND NTFS permissions (most restrictive wins)
  • ABE (Access-Based Enumeration): Users only see folders they have access to — enable per share
  • DFS-R backlog: Normal if <100 files; investigate if growing: bandwidth, conflict, staging area full
  • VSS snapshots failing: Check disk space on shadow copy volume, max size setting
  • FSRM quotas/screening: Use for controlling storage, blocking executables in user shares

SECTION 4 – HYPER-V

4.1 Hyper-V Health Check

# VM status
Get-VM | Select Name, State, CPUUsage, MemoryAssigned, Uptime
Get-VM | Where {$_.State -eq 'Running'} | Get-VMIntegrationService

# Check virtual switches
Get-VMSwitch
Get-VMNetworkAdapter -All

# Check storage
Get-VHD -Path "C:\VMs\disk.vhdx"
Get-VM | Get-VMHardDiskDrive | Select VMName, Path

# Check replication (if Hyper-V Replica configured)
Get-VMReplication
Measure-VMReplication

4.2 Hyper-V Best Practices

  • Use Generation 2 VMs for UEFI/Secure Boot support
  • Dynamic memory: Set startup RAM, min/max range
  • Use VHDX (not VHD) — supports up to 64TB
  • Separate management OS traffic from VM traffic (dedicated NICs)
  • Enable Integration Services on all VMs
  • Regular checkpoint cleanup — checkpoints are NOT backups
  • Reserve 10-15% host resources for management OS

4.3 Hyper-V Troubleshooting

  • VM won't start: Check available resources, VHD path, config version compatibility
  • Slow VM performance: Check overcommit ratio, disk I/O (use VHDX on SSD), network saturation
  • Network connectivity: Check virtual switch binding, VLAN tagging, MAC spoofing settings
  • Live Migration failure: Check credentials, network, compatible processor, shared storage

# Check quorum
Get-ClusterQuorum

# Check CSV health
Get-ClusterSharedVolume
Get-ClusterSharedVolumeState

5.2 Quorum Models

NodesRecommended Witness
2 nodesFile Share Witness or Cloud Witness (mandatory)
3-4 nodesFile Share Witness or Cloud Witness (recommended)
5+ nodes (odd)No witness needed
5+ nodes (even)File Share Witness or Cloud Witness

5.3 Cluster Troubleshooting

  • Split-brain: Quorum witness prevents — always configure a witness
  • Failover not working: Check preferred owner, failback settings, resource dependencies
  • CSV offline: Check disk health, redirected access mode, ownership
  • Network partition: Ensure separate heartbeat network, check switch config

8.2 Common RDS Issues

  • Licensing grace period expired: Properly configure RD Licensing, point session hosts to license server
  • "CredSSP encryption oracle" error: Update both client and server, or set registry AllowEncryptionOracle=2 (temporary)
  • Slow sessions: Check bandwidth, disable visual effects, check anti-virus exclusions
  • Printer redirection failing: Check Easy Print driver, GPO settings for printer redirection
  • Drive redirection blocked: Check GPO: Computer Config > Admin Templates > RDS > Device and Resource Redirection
