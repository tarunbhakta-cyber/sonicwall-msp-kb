1.6 NTDS Database Maintenance

# Offline defrag (requires DSRM boot)
ntdsutil
  activate instance ntds
  files
  compact to C:\temp
  quit
  quit
# Then copy compacted ntds.dit back and delete old logs

  quit
  quit

SECTION 2 – GROUP POLICY

2.1 GPO Processing Order

  1. Local Policy
  2. Site
  3. Domain
  4. OU (parent to child)

Last applied wins. Enforced GPOs override Block Inheritance.

2.2 GPO Troubleshooting

# Log: C:\Windows\debug\usermode\gpsvc.log

2.3 Common GPO Issues

  • GPO not applying: Check security filtering, WMI filter, link enabled, OU placement
  • Slow logon: Check GPO processing time in event log (Event ID 4016), reduce mapped drives/printers
  • Loopback processing: Replace mode replaces user GPOs, Merge mode combines them
  • Preference items failing: Check item-level targeting, remove "Apply once" if needed
  • SYSVOL replication delay: Check DFS-R health: dfsrdiag pollad

2.4 Security Filtering vs WMI Filtering

  • Security Filtering: Controls WHO gets the GPO (user/computer/group). Remove "Authenticated Users" and add specific group.
  • WMI Filtering: Controls WHERE based on system properties (OS version, disk space, etc.)
# WMI filter example: Only Windows 11
SELECT * FROM Win32_OperatingSystem WHERE Version LIKE "10.0.22%" AND ProductType = "1"

SECTION 3 – AZURE AD / ENTRA ID

3.1 Hybrid Identity Architecture

  • Azure AD Connect: Syncs on-prem AD to Azure AD. Password Hash Sync (PHS) recommended as primary.
  • Pass-through Authentication (PTA): Validates passwords against on-prem AD in real-time.
  • Federation (ADFS): Complex, use only if required by compliance. Moving away from this.
  • Cloud-only: No on-prem dependency. Best for born-in-cloud orgs.

3.3 Conditional Access Policies

  • Always exclude break-glass accounts from all CA policies
  • Require MFA for all users, all cloud apps
  • Block legacy authentication
  • Require compliant device for sensitive apps
  • Named locations for trusted networks
  • Session controls for unmanaged devices

4.2 ADFS Common Issues

  • Token signing cert expired: Update cert, export to relying parties
  • WAP connectivity: Check trust between WAP and ADFS farm
  • Claims rules wrong: Test with claims-aware test app, review issuance rules
  • SSL cert mismatch: Ensure SAN covers fs.domain.com and enterpriseregistration.domain.com

SECTION 5 – CERTIFICATE SERVICES (AD CS)

5.1 PKI Health Check

# Check CA health
certutil -CA
certutil -CAInfo

# Check CRL distribution
certutil -verify -urlfetch certificate.cer

# Check certificate templates
certutil -template

# Check pending requests
certutil -view -restrict "disposition=9"

5.2 Common Certificate Issues

  • CRL expired: Republish CRL: certutil -CRL
  • Template not available: Check permissions, CA enrollment services, template replication
  • Auto-enrollment failing: Check GPO, template permissions, CA accessibility
  • LDAPS not working: DC needs a cert with Server Authentication EKU and DC FQDN in SAN

SECTION 6 – L3 ESCALATION DECISION TREE

6.1 AD Authentication Failures

  1. Check account lockout: Get-ADUser -Identity user -Properties LockedOut,lockoutTime
  2. Check time sync: w32tm /query /status (Kerberos needs <5 min skew)
  3. Check DC connectivity: nltest /dsgetdc:domain.local
  4. Check DNS resolution: nslookup _ldap._tcp.dc._msdcs.domain.local
  5. Check Kerberos: klist then klist purge and retry
  6. Check Secure Channel: Test-ComputerSecureChannel -Verbose
  7. If secure channel broken: Test-ComputerSecureChannel -Repair -Credential (Get-Credential)

6.2 Replication Emergency

  1. Identify scope: repadmin /replsummary
  2. Check DNS: All DCs resolve each other by FQDN
  3. Check RPC: portqry -n DC02 -e 135 -p tcp
  4. Check firewall: All AD ports open between DCs
  5. Force replication: repadmin /syncall /AdeP
  6. If tombstone lifetime exceeded (60/180 days): Do NOT reconnect — metadata cleanup required
