Microsoft L3 Support – Microsoft 365 Administration

Scope: Exchange Online, SharePoint Online, Teams, OneDrive, Intune/Endpoint Manager, Purview, Defender for O365

SECTION 1 – EXCHANGE ONLINE

1.1 Exchange Online PowerShell

# Connect
Connect-ExchangeOnline -UserPrincipalName admin@domain.com

# Mailbox diagnostics
Get-Mailbox -Identity user@domain.com | FL
Get-MailboxStatistics -Identity user@domain.com
Get-MailboxPermission -Identity user@domain.com
Get-RecipientPermission -Identity user@domain.com  # Send-As

# Message trace (last 10 days)
Get-MessageTrace -SenderAddress user@domain.com -StartDate (Get-Date).AddDays(-7) -EndDate (Get-Date)

# Historical message trace (10-90 days)
Start-HistoricalSearch -ReportTitle "Search1" -StartDate "03/01/2026" -EndDate "03/04/2026" -ReportType MessageTrace -SenderAddress user@domain.com

1.2 Mail Flow Troubleshooting

  • NDR 550 5.1.1: Recipient doesn't exist. Check address, aliases, mail-enabled group membership
  • NDR 550 5.7.1: Relay denied. Check accepted domains, connector config, SPF
  • Stuck in Outbox: Check message size limits, shared mailbox send-as permissions, Outlook profile
  • Delayed delivery: Check transport rules, connectors, partner org throttling
  • External not receiving: Check SPF, DKIM, DMARC, MX records, connector restrictions

1.3 Email Authentication (SPF/DKIM/DMARC)

# SPF Record (DNS TXT)
v=spf1 include:spf.protection.outlook.com -all

# DKIM — Enable via Exchange Admin Center or PowerShell
New-DkimSigningConfig -DomainName domain.com -Enabled $true
Get-DkimSigningConfig -Identity domain.com

# DMARC Record (DNS TXT at _dmarc.domain.com)
v=DMARC1; p=quarantine; rua=mailto:dmarc@domain.com; pct=100

# Validate records
nslookup -type=txt domain.com
nslookup -type=txt selector1._domainkey.domain.com
nslookup -type=txt _dmarc.domain.com

SECTION 2 – SHAREPOINT ONLINE

2.1 SharePoint Administration

# Connect
Connect-SPOService -Url https://domain-admin.sharepoint.com

# Check site collections
Get-SPOSite -Limit All | Select Url, StorageUsageCurrent, Template

# Check permissions
Get-SPOSiteGroup -Site https://domain.sharepoint.com/sites/Finance

# Storage management
Get-SPOTenant | Select StorageQuota, StorageQuotaAllocated

# External sharing settings
Get-SPOTenant | Select SharingCapability
# Values: Disabled, ExternalUserSharingOnly, ExternalUserAndGuestSharing, ExistingExternalUserSharingOnly

2.2 Common SharePoint Issues

  • Sync errors (OneDrive client): Check file name/path length (400 char limit), unsupported characters, file size (250GB max)
  • Permission inheritance broken: Check unique permissions on library/folder/item level
  • "Access Denied" after migration: Recheck permissions, clear browser cache, check conditional access
  • Versioning eating storage: Set version limits (recommend 50-100 major versions)
  • Search not returning results: Check crawl status, managed properties, content source

SECTION 3 – MICROSOFT TEAMS

3.1 Teams Administration

# Connect
Connect-MicrosoftTeams

# Check team info
Get-Team -DisplayName "IT Support"
Get-TeamUser -GroupId 

# Check Teams policies
Get-CsTeamsMeetingPolicy
Get-CsTeamsMessagingPolicy
Get-CsTeamsCallingPolicy

# Check PSTN/calling
Get-CsOnlineUser -Identity user@domain.com | Select LineURI, EnterpriseVoiceEnabled, TeamsUpgradeEffectiveMode

3.2 Teams Troubleshooting

  • Call quality issues: Check Teams Admin Center > Call Analytics, run network assessment tool
  • Can't create teams: Check M365 group creation policy (Azure AD > Group settings)
  • Guest access not working: Check Teams admin > Org-wide settings > Guest access, Azure AD external collaboration settings
  • Federation issues: Check Teams admin > External access, DNS SRV records for federation
  • Teams phone issues: Check voice routing policies, dial plans, PSTN connectivity (Direct Routing/Calling Plans)

SECTION 4 – MICROSOFT INTUNE / ENDPOINT MANAGER

4.1 Device Enrollment

  • Windows Autopilot: Zero-touch deployment. Requires hardware hash uploaded to Intune.
  • Azure AD Join: Cloud-only identity. Settings > Accounts > Work or School > Join Azure AD.
  • Hybrid Azure AD Join: Device joined to on-prem AD AND Azure AD. Requires Azure AD Connect.
  • iOS/Android: Company Portal app or managed Apple IDs / Android Enterprise.

4.2 Intune Troubleshooting

# Check device compliance
Get-MgDeviceManagementManagedDevice -Filter "userPrincipalName eq 'user@domain.com'"

# On the device:
dsregcmd /status   # Check Azure AD join / hybrid join status
mdmdiagnosticstool.exe -area DeviceEnrollment;DeviceProvisioning;Autopilot -zip C:\temp\intune-diag.zip

# Check Intune management extension log
C:\ProgramData\Microsoft\IntuneManagementExtension\Logs\IntuneManagementExtension.log

# Sync device
Invoke-IntuneManagedDeviceSyncDevice

4.3 Common Intune Issues

  • Policy not applying: Check assignment groups, conflict resolution, sync status
  • App install failing: Check detection rules, requirements, dependency chain, install logs
  • Compliance policy stuck: Check grace period, device sync, conditional access blocking
  • Autopilot ESP timeout: Check network, required apps, profile assignment, enrollment restrictions
  • Win32 app detection: Use registry key, file/folder existence, or MSI product code

SECTION 5 – ONEDRIVE FOR BUSINESS

5.1 OneDrive Troubleshooting

  • Sync pending: Check OneDrive settings > About for version, reset sync: %localappdata%\Microsoft\OneDrive\onedrive.exe /reset
  • Known Folder Move (KFM) not working: Check GPO/Intune policy, verify tenant ID, user not redirected via legacy folder redirection
  • Storage full: Check admin center for storage quota, retention policies holding deleted files
  • Files On-Demand issues: Check NTFS volume, Windows 10 1709+, setting enabled in OneDrive

SECTION 6 – MICROSOFT DEFENDER FOR OFFICE 365

6.1 Protection Stack

  1. Connection filtering (IP allow/block)
  2. Anti-malware (EOP)
  3. Mail flow rules
  4. Anti-spam (EOP)
  5. Anti-phishing (Defender for O365)
  6. Safe Attachments (sandbox detonation)
  7. Safe Links (URL rewriting/scanning)

6.2 Common Protection Issues

  • Legitimate email quarantined: Check quarantine reason, add to allow list / transport rule, submit false positive
  • Phish getting through: Verify Safe Attachments/Links policies, check impersonation protection settings
  • Alert fatigue: Tune alert policies in M365 Defender portal, use automated investigation
