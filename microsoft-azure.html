SECTION 1 – AZURE VIRTUAL MACHINES

1.1 VM Troubleshooting

# Check VM status
az vm get-instance-view --resource-group RG01 --name VM01 --query instanceView.statuses

# Serial console (when RDP/SSH fails)
# Azure Portal > VM > Help > Serial Console

# Boot diagnostics
az vm boot-diagnostics get-boot-log --resource-group RG01 --name VM01

# Redeploy VM (moves to new host)
az vm redeploy --resource-group RG01 --name VM01

# Reset RDP access
az vm user reset --resource-group RG01 --name VM01 --username adminuser --password NewP@ssw0rd

# Run command remotely
az vm run-command invoke --resource-group RG01 --name VM01 --command-id RunPowerShellScript --scripts "Get-Service"

1.3 Common VM Issues

  • VM stuck "Starting": Check allocation failures (resize or move to different region)
  • RDP timeout: Check NSG rules (port 3389), Windows Firewall, NIC effective security rules
  • High disk latency: Check disk type (Standard HDD vs Premium SSD), IOPS limits, disk caching
  • VM not booting: Check boot diagnostics screenshot, serial console, repair VM using OS disk swap
  • Unexpected reboot: Check Azure Service Health, VM availability logs, planned maintenance

SECTION 2 – AZURE NETWORKING

2.1 VNet Architecture

  • Hub-spoke topology recommended for multi-VNet designs
  • Address spaces must NOT overlap between peered VNets or VPN-connected networks
  • Subnets: GatewaySubnet (/27 minimum), AzureFirewallSubnet (/26), AzureBastionSubnet (/26)
  • NSGs applied at subnet level (recommended) or NIC level
  • UDRs (Route Tables) for forced tunneling or NVA routing

2.2 Network Security Groups (NSG)

# Check effective NSG rules
az network nic show-effective-nsg --resource-group RG01 --name VM01-NIC

# Check NSG flow logs
az network watcher flow-log show --resource-group NetworkWatcherRG --watcher-name NetworkWatcher --name NSGFlowLog

# Common NSG troubleshooting
# Azure Portal > VM > Networking > Effective security rules
# Use IP flow verify: Network Watcher > IP flow verify

2.5 Site-to-Site VPN Troubleshooting

  • Phase 1 (IKE) failure: Check pre-shared key match, IKE version, encryption/hash mismatch
  • Phase 2 (IPsec) failure: Check traffic selectors, PFS setting, SA lifetime mismatch
  • Tunnel up but no traffic: Check UDRs, NSGs, on-prem firewall rules, address space in local network gateway
  • Intermittent drops: Check DPD (Dead Peer Detection) settings, MTU/fragmentation, ISP stability

SECTION 4 – AZURE BACKUP

4.1 Backup Troubleshooting

# Check backup job status
az backup job list --resource-group RG01 --vault-name RSV01 --query "[?status=='Failed']"

# Check protected items
az backup item list --resource-group RG01 --vault-name RSV01

# Trigger on-demand backup
az backup protection backup-now --resource-group RG01 --vault-name RSV01 --container-name VM01 --item-name VM01

4.2 Common Backup Issues

  • VSS snapshot failure: Check VSS writers: vssadmin list writers, restart failed writers
  • Agent not responding: Check MARS agent service, network to *.backup.windowsazure.com (443)
  • VM backup failing: Check VM agent status, snapshot extension, disk lock
  • Restore point consistency: Application-consistent requires VSS; crash-consistent if VSS fails

SECTION 5 – AZURE SITE RECOVERY (ASR)

5.1 ASR Health Check

# Check replication health
az site-recovery replication-protected-item list --resource-group RG01 --vault-name RSV01 --fabric-name primary --protection-container-name pcprimary

# Check RPO
# Azure Portal > Recovery Services Vault > Replicated Items > RPO column
# RPO > 30 min = investigate network/disk throughput

# Failover test (ALWAYS test monthly)
# Portal > Replicated item > Test Failover > Select recovery point > Select target VNet

5.2 ASR Common Issues

  • Initial replication stuck: Check mobility service agent, port 9443 (process server), disk throughput
  • RPO breach: Check source VM disk I/O, network bandwidth to Azure, process server health
  • Failover failed: Check target resource availability, NSG rules, storage account access

SECTION 6 – AZURE MONITOR & LOG ANALYTICS

6.1 Key Monitoring Queries (KQL)

// VM CPU spikes last 24h
Perf
| where ObjectName == "Processor" and CounterName == "% Processor Time"
| where TimeGenerated > ago(24h)
| summarize avg(CounterValue) by Computer, bin(TimeGenerated, 5m)
| where avg_CounterValue > 80

// Failed sign-ins
SigninLogs
| where ResultType != "0"
| summarize count() by UserPrincipalName, ResultDescription, AppDisplayName
| order by count_ desc

// Disk space alerts
Perf
| where ObjectName == "LogicalDisk" and CounterName == "% Free Space"
| where CounterValue < 10
| summarize arg_min(TimeGenerated, CounterValue) by Computer, InstanceName

6.2 Alert Best Practices

  • CPU sustained >85% for 15 min = Warning
  • Available memory <10% for 10 min = Critical
  • Disk free <15% = Warning, <5% = Critical
  • VM heartbeat missing for 5 min = Critical
  • Backup job failed = Critical (immediate notification)
