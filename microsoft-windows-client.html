Microsoft L3 Support – Windows Client Troubleshooting

Scope: Windows 10/11, Deployment, Drivers, Networking, Performance, Printing, Profiles, BitLocker, Updates

SECTION 1 – BOOT & STARTUP ISSUES

1.1 Boot Failure Diagnosis

# Access WinRE (Windows Recovery Environment)
# Hold Shift + Restart, or boot from USB

# Repair boot configuration
bootrec /fixmbr
bootrec /fixboot
bootrec /scanos
bootrec /rebuildbcd

# UEFI boot repair
bcdboot C:\Windows /s S: /f UEFI

# Safe Mode via command line (from WinRE)
bcdedit /set {default} safeboot minimal    # Safe Mode
bcdedit /set {default} safeboot network    # Safe Mode with Networking
# Remove after: bcdedit /deletevalue {default} safeboot

# System File Checker
sfc /scannow
# If SFC fails:
DISM /Online /Cleanup-Image /RestoreHealth
sfc /scannow   # Run again after DISM

1.2 BSOD Analysis

  • Memory dump location: C:\Windows\Minidump\ or C:\Windows\MEMORY.DMP
  • Analyze with WinDbg: !analyze -v
  • Common stop codes:
Stop CodeCommon CauseAction
IRQL_NOT_LESS_OR_EQUALBad driverUpdate/rollback driver, check RAM
KERNEL_DATA_INPAGE_ERRORDisk failureCheck disk health, run chkdsk
SYSTEM_SERVICE_EXCEPTIONDriver/software conflictBoot safe mode, uninstall recent software
CRITICAL_PROCESS_DIEDCorrupted system filesSFC /scannow, DISM repair

netsh interface ip show config

netsh wlan show interfaces

# Export WiFi report:
netsh wlan show wlanreport

# DNS resolution
nslookup domain.local
Resolve-DnsName -Name dc01.domain.local -Type A

# Port connectivity test
Test-NetConnection -ComputerName server.domain.local -Port 443
tnc 192.168.1.1 -Port 3389

2.2 VPN Client Troubleshooting

  • Always On VPN: Check device tunnel vs user tunnel, profile XML, NPS policies, certificate
  • Split tunnel vs force tunnel: Check route table during VPN (route print)
  • DNS resolution over VPN: Check NRPT (Name Resolution Policy Table) rules
  • SonicWall NetExtender: Check TLS version, adapter driver, route injection
  • GlobalProtect: Check HIP (Host Information Profile), portal/gateway config, cert

SECTION 3 – PERFORMANCE

3.1 Performance Diagnostics

# Quick performance check
Get-Process | Sort CPU -Descending | Select -First 10 Name, CPU, WorkingSet
Get-Counter '\Processor(_Total)\% Processor Time','\Memory\Available MBytes','\PhysicalDisk(_Total)\% Disk Time' -SampleInterval 2 -MaxSamples 5

# Disk health
wmic diskdrive get status,model
Get-PhysicalDisk | Select FriendlyName, MediaType, OperationalStatus, HealthStatus

# Memory diagnostics
# Run: mdsched.exe (Windows Memory Diagnostic)
# Or from cmd: bcdedit /bootsequence {memdiag}

# Performance trace (60 seconds)
perfmon /report   # Generates system diagnostics report

3.2 Slow PC Methodology

  1. Check Task Manager: CPU, Memory, Disk columns. Identify top consumer.
  2. If Disk 100%: Check for failing drive (wmic diskdrive get status), disable Superfetch/SysMain, check Windows Search indexer
  3. If High CPU: Check process (antivirus scan, Windows Update, WMI Provider Host)
  4. If Low Memory: Check commit charge, pagefile size, identify memory leak process
  5. Check startup items: Task Manager > Startup — disable unnecessary items
  6. Check malware: Run offline scan with Defender or MSERT
  7. Check disk space: At least 15% free on C: drive
  8. Check thermals: Use HWiNFO64 for CPU/GPU temps

SECTION 5 – USER PROFILES

5.1 Profile Troubleshooting

# Check profile list
Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList\*" |
    Select PSChildName, ProfileImagePath, State

# Profile states:
# 0 = Profile ready
# 1 = Profile being created
# 2 = Profile being deleted

# Fix "temporary profile" issue:
# 1. Backup user data from C:\Users\username
# 2. Delete profile SID entry from ProfileList registry key
# 3. Delete C:\Users\username folder
# 4. Reboot and log in (fresh profile created)

# Roaming profile issues
# Check: \\server\profiles$\username.V6 (Win 10/11)
# Verify permissions: User = Full Control on their folder
# Check GPO: Computer Config > Admin Templates > System > User Profiles

5.2 Folder Redirection Troubleshooting

  • Check GPO: User Config > Policies > Windows Settings > Folder Redirection
  • Verify target path accessible and permissions correct
  • Check: gpresult /R for applied folder redirection GPO
  • Offline files sync issues: Check CSC cache, reset if needed: CSCCmd.exe /enum

SECTION 6 – PRINTING

6.1 Print Troubleshooting

# Clear print queue
Stop-Service Spooler
Remove-Item C:\Windows\System32\spool\PRINTERS\* -Force
Start-Service Spooler

# Check printer status
Get-Printer | Select Name, PortName, DriverName, PrinterStatus
Get-PrintJob -PrinterName "HP LaserJet"

# Reinstall printer driver
Remove-PrinterDriver -Name "HP Universal Printing PCL 6"
Add-PrinterDriver -Name "HP Universal Printing PCL 6"

# Print Management snap-in
printmanagement.msc

6.2 Common Print Issues

  • Print spooler crashing: Bad driver — isolate driver in separate process, update/replace driver
  • Slow printing: Check spooling setting (print directly vs spool), driver type (PCL vs PS), network path
  • Printer offline: Check port config, SNMP status, WSD vs TCP/IP port
  • Point and Print not working: Check GPO restrictions (PrintNightmare patches), Package Point and Print

SECTION 7 – BITLOCKER

7.1 BitLocker Management

# Check BitLocker status
manage-bde -status
Get-BitLockerVolume

# Enable BitLocker
Enable-BitLocker -MountPoint "C:" -EncryptionMethod XtsAes256 -RecoveryPasswordProtector

# Get recovery key
manage-bde -protectors -get C:
(Get-BitLockerVolume -MountPoint "C:").KeyProtector | Where KeyProtectorType -eq "RecoveryPassword"

# Backup recovery key to AD
Backup-BitLockerKeyProtector -MountPoint "C:" -KeyProtectorId "{GUID}"

# Suspend BitLocker (for BIOS updates)
Suspend-BitLocker -MountPoint "C:" -RebootCount 1

# Unlock with recovery key
manage-bde -unlock C: -RecoveryPassword 123456-789012-345678-901234-567890-123456-789012-345678

7.2 Common BitLocker Issues

  • Recovery key prompt on every boot: Check TPM health (tpm.msc), re-seal BitLocker, check Secure Boot
  • Can't enable: Check TPM 2.0 present and enabled, UEFI boot, GPT partition, Secure Boot
  • Recovery key not in AD: Manually backup: manage-bde -protectors -adbackup C: -ID {GUID}
  • Intune BitLocker policy: Check device compliance, encryption profile, silent enablement requirements
